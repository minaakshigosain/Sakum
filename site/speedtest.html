How it's built

The speed tester consumes the language. Two capabilities it needed did not exist in the core, so — per SAKUM_LANG.md §1.9 ("add necessary library functions if not present in Sakum") — the language grew. Everything is raw x86-64, no host-language layer.

assembly/sakum_lib_dns.s

DNS resolution

sakum_dns_resolve(host) walks getaddrinfo's reply list for the first AF_INET entry and returns the IPv4 in network order — ready to drop into a sockaddr_in. (Previously the crawler only accepted raw IP literals.)

assembly/sakum_lib_time.s

Microsecond clock

sakum_now_us() / sakum_elapsed_us(t0) wrap gettimeofday through a dedicated .bss timeval buffer (a stack buffer gets clobbered on nested calls). Gives the latency deltas the tester reports.

assembly/sakum_speedtest.s

The tester

Opens a BSD socket, connects, sends an HTTP/1.1 GET, and loops recv while snapshotting TTFB at the first byte. Reports DNS / connect / TTFB / total latency + throughput in plain assembly.

request flow (all machine level)
hostname → sakum_dns_resolve → socket() → connect() → send(GET) → recv loop + TTFB → metrics
assembly/sakum_speedtest.s (excerpt)
# resolve + connect, timed
SP_resolve_and_connect:
    call sakum_now_us          # t0
    lea  rsi, [rip + dns_ip]
    call sakum_dns_resolve     # hostname -> IPv4
    call sakum_elapsed_us     # -> t_dns
    # ... socket(); build sockaddr_in; connect() ...
    call sakum_elapsed_us     # -> t_connect

# HTTP GET, snapshot TTFB at first recv byte
recv_loop:
    syscall                     # recv(sock, buf, 4096, 0)
    cmp  rax, 0
    jle  recv_done
    test r14, r14
    jnz  no_ttfb
    call sakum_elapsed_us     # -> t_ttfb (first byte)
    inc  r14
no_ttfb:
    add  r13, rax               # accumulate bytes

Same idea, in Sakum source

The high-level intent lives in examples/speedtest.sak. The front end lowers this through the SIR to the x86-64 above. 

# speedtest.sak — high-level intent of the asm tester
kriya speedtest(host) {
    naam t0   = sakum_now_us();
    naam tdns = time_us({ dns_resolve(host); });
    naam sock = SP_resolve_and_connect(host, 80);
    naam ttb  = sakum_now_us();
    naam bytes= SP_http_get(sock, "/");
    naam ttfb = sakum_elapsed_us(ttb);
    naam ttot = sakum_elapsed_us(t0);
    naam spd  = bytes * 1000000 / ttot;
    lek("DNS: " + tdns + " us, TTFB: " + ttfb);
    lek("Throughput: " + spd + " bytes/s");
}
speedtest("example.com");

What building it taught the language

DNS is now a library primitive

Hostnames resolve through a real, reusable asm function — the crawler and any future net tool share it.

Timing is first-class

Microsecond now/elapsed calls let any routine measure itself without libc timers.

macOS quirks documented

clock_gettime isn't a raw syscall here; a .bss timeval buffer avoids stack clobber on nested calls. Recorded in memory.md.

Graceful failure

Bad hosts return a clean speedtest failed instead of a segfault — failure modes are part of the design.
